# 下载管理器 UI/UX 设计文档

> 参考百度网盘、迅雷、IDM 等主流下载工具的设计规范

---

## 一、页面结构

### 1. 首页（HomeScreen）
- 平台选择卡片（抖音/小红书）
- 底部显示最近下载概要（可选）

### 2. 平台页面（PlatformShell）
- **底部导航栏**：下载 / 任务 / 设置
- **下载页**：链接输入 + 快捷操作
- **任务页**：下载任务列表 + 进度
- **设置页**：Cookie / 缓存清理 / 关于

## 三、任务页设计（TasksScreen）— 核心页面

### 布局（参考百度网盘/迅雷）
```
┌─────────────────────────────────┐
│ 下载任务 (3)          [⋮ 更多]   │  ← 标题栏
├─────────────────────────────────┤
│                                  │
│ ┌─────────────────────────────┐ │
│ │ 📹 视频  作者名_标题          │ │  ← 任务卡片
│ │ ✅ 已完成  12.3 MB            │ │
│ │ ━━━━━━━━━━━━━━━━━━━━ 100%   │ │  ← 进度条
│ │                    [删除]     │ │
│ └─────────────────────────────┘ │
│                                  │
│ ┌─────────────────────────────┐ │
│ │ 🖼️ 图集  作者名_标题          │ │
│ │ ⏬ 下载中  3/10 张             │ │  ← 图集进度
│ │ ━━━━━━━━━░░░░░░░░░░░ 30%    │ │
│ │              [⏸暂停] [❌]    │ │
│ └─────────────────────────────┘ │
│                                  │
│ ┌─────────────────────────────┐ │
│ │ 📹 视频  作者名_标题          │ │
│ │ ❌ 失败: Cookie过期            │ │
│ │              [🔄重试] [❌]    │ │
│ └─────────────────────────────┘ │
│                                  │
├─────────────────────────────────┤
│ [⏸全部暂停] [▶全部继续] [🗑清理] │  ← 底部操作栏
└─────────────────────────────────┘
```

### 任务卡片规范
- **图标**：左侧显示类型图标（📹视频/🖼️图集/🎥直播）
- **标题**：`作者名_作品标题`
- **状态行**：
  - 解析中：`解析中...`
  - 下载中（有大小）：`下载中 45% (2.1MB/4.6MB)`
  - 下载中（图集）：`下载中 3/10 张`
  - 已完成：`已完成 12.3 MB`
  - 已暂停：`已暂停 (2.1MB/4.6MB)`
  - 失败：`失败: 错误信息`
- **进度条**：下载中/暂停时显示
- **操作按钮**（右侧）：
  - 下载中 → `[⏸暂停]`
  - 暂停/失败 → `[▶继续]` `[🗑删除]`
  - 已完成 → `[📂打开]` `[🗑删除]`

### 底部操作栏规范
- **仅在有任务时显示**
- 使用 `BottomAppBar` 或固定底部栏
- 按钮：
  - `[⏸ 全部暂停]` — 暂停所有下载中任务
  - `[▶ 全部继续]` — 继续所有暂停/失败任务  
  - `[🗑 清理已完成]` — 清除已完成任务
- 按钮在无对应操作时置灰（disabled）

### 空状态
```
        📥
    暂无下载任务
    
  去下载页粘贴链接开始下载
```

## 四、进度逻辑规范

### 下载流程状态机
```
用户点击下载
    ↓
[解析中] — 创建任务，状态=downloading，标题="解析中: URL"
    ↓
解析成功，开始下载文件
    ↓
[下载中] — 实时更新 downloadedSize/totalSize
    ↓ (成功)
[已完成] — 更新标题为实际标题，设置 filePath/totalSize
    ↓ (失败)
[失败] — 设置 errorMessage
    ↓ (用户暂停)
[已暂停]
```

### 进度更新频率
- 每 64KB 更新一次（避免过于频繁刷新 UI）
- 使用 `DownloadTaskManager.updateProgress()` 而非 `updateTask()`（更轻量）

### 图集下载进度
- 图集进度 = 已下载图片数 / 总图片数
- 状态显示：`下载中 3/10 张`
- 完成后扫描目录获取总大小

## 六、交互规范

### SnackBar 规范
- **时长**：3秒
- **格式**：`标题: 消息内容`
- **成功**：绿色背景或图标
- **失败**：红色背景或图标

### Dialog 规范（仅在必要时使用）
- **删除确认**：删除任务/清理缓存时
- **任务详情**：点击任务卡片查看详细信息
- **Cookie 输入**：手动输入 Cookie

### 跳转规范
- 下载开始后 → 自动跳转任务页（可选）
- 任务页返回 → 回到下载页
- 平台页返回 → 回到首页
